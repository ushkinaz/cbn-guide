# Color Usage Rules

This project uses two distinct sets of colors that must NOT be mixed or used interchangeably.

## 1. Game Colors (Read-Only / Generated)

These colors are pulled directly from the game source execution and represent valid game colors (e.g., for map tiles, items, monsters).

- **Source**: Generated by `@[generate-color-css.js]`.
- **Prefix**: All CSS variables are prefixed with `--cata-color-` (e.g., `--cata-color-red`, `--cata-color-cyan`).
- **Usage**:
    - Use these ONLY when rendering game entities or imitating game terminal output.
    - **NEVER** use these colors for the web application UI (buttons, navigation, headers, etc.).
    - **NEVER** manually modify the values of these variables in the CSS. If an update is needed, modify `generate-color-css.js` and regenerate the CSS.

## 2. Web App Colors

These colors are used for the website interface, intended to provide a modern, distinct look separate from the game's retro terminal aesthetic.

- **Source**: Defined manually in CSS files (e.g., `public/global.css`).
- **Characteristics**: Typically use HSL, HEX, or standard RGB values (e.g., `hsl(185deg, 45%, 45%)`, `#333`, `#1c62b9`).
- **Usage**:
    - Use these for all UI components: buttons, links, layout containers, forms, typography.
    - **NEVER** use `--cata-color-` variables for these elements.

## Summary Table
